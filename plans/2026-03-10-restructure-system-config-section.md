# Raspberry Pi PXE + iSCSI 記事「システム構成」セクション整理計画

## Context

「# システム構成」セクション内の「## ハードウェア」と「## アーキテクチャ図」の区別がわかりづらい。

具体的な問題:
- 「ハードウェア」サブセクションに物理機器テーブルと論理ストレージマッピング図（Mermaid）が同居している
- 「アーキテクチャ図」は別の Mermaid 図でソフトウェアサービス構成を示すが、NAS・Pi・接続関係は「ハードウェア」の図と重複している
- 2 つの Mermaid 図が何を示しているのか、見出しだけでは判別できない

## 修正方針

2 つの図は関心事が異なる（ストレージ構造 vs ネットワーク・サービス構成）ため、**図の統合は行わず**、見出しリネームとキャプション追加で役割を明確化する。

統合を見送る理由:
- 統合するとノード数が爆発し（Pi 3 台 × 2 マウントポイント）、線の交差で読みにくくなる
- モバイル幅では特に致命的
- 既存図を変更しないためレンダリング崩れのリスクがない

## 提案する新構成

```
# システム構成
  （冒頭の説明文はそのまま）

  ## 構成機器                    ← 旧「ハードウェア」からリネーム
    - ハードウェアテーブル（現状維持）
    - ダミー値の注意書き（現状維持）
    ※ Mermaid 図はここから除去

  ## ストレージ構成              ← ★新サブセクション（旧ハードウェア内の Mermaid 図を移動）
    - キャプション追加
    - Mermaid 図 1（NAS 内部ストレージ + Pi マウント関係、現状維持）

  ## ネットワーク・サービス構成  ← 旧「アーキテクチャ図」からリネーム
    - キャプション追加
    - Mermaid 図 2（サービス構成 + ネットワーク接続、現状維持）
    - NFS 補足の :::message（現状維持）

  ## ブートシーケンス（現状維持）
```

## 具体的な変更内容

### 変更 A: 「## ハードウェア」→「## 構成機器」にリネーム、Mermaid 図を分離
- テーブルと注意書きのみを残す
- Mermaid 図は次の新サブセクションへ移動

### 変更 B: 「## ストレージ構成」サブセクション新設
- 構成機器と旧アーキテクチャ図の間に挿入
- キャプション: 「NAS 内部のストレージ配置と、Pi のマウント先の対応関係を示します（代表として Pi #1 のみ記載）。」
- Mermaid 図 1 をここに配置（図自体は変更なし）

### 変更 C: 「## アーキテクチャ図」→「## ネットワーク・サービス構成」にリネーム
- キャプション追加: 「NAS 上で動作するサービスと、各 Pi・ルーターとのネットワーク接続関係を示します。」
- Mermaid 図 2 と NFS 補足の :::message はそのまま
