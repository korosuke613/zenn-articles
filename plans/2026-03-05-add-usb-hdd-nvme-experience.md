# USB HDD 運用経験と NVMe 見送りの経緯を追加

## Context

記事の「なぜネットワークブートか」セクションでは、SD カードの課題からいきなりネットワークブートの利点に飛んでいる。実際にはその間に USB 外付け HDD での運用を試みた経験があり、さらに NVMe も検討したが見送った経緯がある。この遍歴を追加することで、PXE + iSCSI を選択した動機がより説得力を持つ。

## 修正内容

### 対象ファイル
- `articles/raspberry-pi-pxe-iscsi-boot.md`

### 変更 1: 新セクション追加（L35-36 付近）

「## SD カードの課題」の後、「## ネットワークブートの利点」の前に追加。

```markdown
## 他のブート方式の検討

SD カードの課題を受けて、まず外付け HDD を USB 接続（micro USB Type-B）で運用していました。しかし、micro USB Type-B の接触不良が発生しやすく、Pi を移動するたびにケーブルが抜けるリスクがありました。各 Pi に HDD とケーブルがぶら下がる状態も取り回しが悪く、長期運用には向いていませんでした。

NVMe ブートも検討しましたが、Raspberry Pi 4B で NVMe を利用するには各 Pi に NVMe HAT（または USB-NVMe アダプタ）と NVMe SSD を追加購入する必要があり、3 台分のコストを考えて見送りました。

これらの経験を踏まえて、ストレージを NAS に集約するネットワークブート（PXE + iSCSI）に行き着きました。
```

### 変更 2: 「良かった点」に接続信頼性の項目を追加（L627 付近）

「SD カードの心配から解放された」はそのまま維持。「パフォーマンスの向上」の後に新項目を追加:

```markdown
- **接続の信頼性向上**: USB HDD 運用時代は micro USB Type-B の接触不良に悩まされていたが、ネットワークブートでは Ethernet ケーブル（RJ45）のみで完結する。RJ45 はラッチ付きで抜けにくく、接続の信頼性が格段に高い
```

### 変更 3: 「周辺機器・ケーブルの削減」の表現調整（L42 付近）

現状:
```markdown
- **周辺機器・ケーブルの削減**: USB SSD ブートと比べて、各 Pi に USB ケーブルや外付け SSD を接続する必要がない。物理的な接続点が減ることで、ケーブル抜けや接触不良による障害リスクも下がる
```

修正後（USB SSD → USB ストレージに表現を広げる）:
```markdown
- **周辺機器・ケーブルの削減**: USB ブートと比べて、各 Pi に USB ケーブルや外付けストレージを接続する必要がない。物理的な接続点が減ることで、ケーブル抜けや接触不良による障害リスクも下がる
```
